# ADR 0001: 以 PostgreSQL + JSONB + pgvector 作为 Phase 1 主存储

## 状态

Accepted

## 背景

Phase 1 需要同时满足这些需求：

- 存储脚本主记录与异步状态
- 存储拆解后的结构化片段
- 存储标签、生成日志和点评结果
- 支持基于 embedding 的相似检索
- 保持部署简单，不引入过多基础设施

## 决策

Phase 1 使用 PostgreSQL 作为单一运行时主存储：

- 结构化字段直接建表
- 非固定结构信息使用 JSONB
- 向量字段使用 pgvector
- 数据结构演进通过 Flyway 管理

## 结果

正面影响：

- 数据一致性边界清晰
- 部署形态简单，适合 MVP 迭代
- 生成日志、标签和脚本详情可统一查询
- 能在一个存储中完成大部分检索和聚合

负面影响：

- 高维向量检索的性能优化空间受 pgvector 和部署环境限制
- `test` 环境无法完全模拟 PostgreSQL 特性
- 后续如果进入大规模检索场景，可能需要拆出专用向量方案

## 备选方案

### PostgreSQL + 外部向量数据库

优点：

- 更适合高规模语义检索

缺点：

- MVP 复杂度明显上升
- 数据同步和回写链路更复杂

### 文档数据库或搜索引擎为主库

优点：

- 非结构化内容检索灵活

缺点：

- 事务与关系约束更弱
- 对当前业务主链路并不比 PostgreSQL 更直接
